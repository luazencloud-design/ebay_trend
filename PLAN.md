# eBay K-상품 랭킹 트렌드 사이트 기획서

> **한 줄 정의**: eBay에서 잘 팔리는 한국 상품 카테고리 → 브랜드 → 상품 → 소싱처 → 추천 판매 상품까지, 셀러의 의사결정을 한 흐름으로 연결하는 리서치 대시보드.

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| **타겟 시장** | eBay (글로벌) |
| **타겟 사용자** | 개인 셀러, 1인 무역 사업자, 부업 셀러, 무역창업 수강생 |
| **차별점** | 1~5위 레드오션 회피 + 6~20위 블루오션 발굴 + 소싱처 직결 |
| **리서치 주체** | Gemini (하루 1회 배치 결과물 제공) |
| **사이트 역할** | Gemini가 만든 데이터를 시각화/탐색 가능하게 표시 |

---

## 2. 핵심 사용자 흐름 (3-Depth Drill-down)

```
[1단계] 카테고리 랭킹 (메인 화면)
    │
    ↓ 카테고리 클릭
    │
[2단계] 카테고리 상세 (브랜드 + 상품 + 추천 판매 사이트)
    │
    ↓ 판매 사이트 클릭
    │
[3단계] 해당 사이트에서 팔기 좋은 상품 리스트
```

---

## 3. 화면별 상세 기획

### [1단계] 메인 — 카테고리 랭킹

**표시 정보**
- eBay에서 한국 상품이 잘 팔리는 카테고리 TOP 20
- 카드 형태로 그리드 배치

**카드 구성 요소**
- 카테고리명 (예: Beauty, K-Stationery, Fashion 등)
- 현재 순위
- 순위 변동 (▲ 상승 / ▼ 하락 / ─ 유지)
- 경쟁강도 (★ 1~5단계)
- 평균 마진율
- 레드/블루 존 배지 (1~5위: 빨강, 6~20위: 파랑)

**필터/정렬**
- 기간: 2025 / 2026
- 지역: US / EU / 글로벌
- 정렬: 순위순 / 마진 높은순 / 경쟁 낮은순

---

### [2단계] 카테고리 상세 — 3분할 레이아웃

선택한 카테고리에 대한 모든 정보를 한 페이지에서 확인.

| 좌측 (브랜드) | 중앙 (상품) | 우측 (소싱처) |
|---|---|---|
| **브랜드 랭킹 TOP 20** | **상품 랭킹 TOP 30** | **추천 도매/소싱 사이트 TOP 10** |
| 브랜드명, 순위, 변동 | 상품명, 순위, eBay 평균가 | 사이트명, 신뢰도, 카테고리 적합도 |
| 클릭 시 → 해당 브랜드 상품만 필터 | 클릭 시 → eBay 리스팅 상세 | **클릭 시 → 3단계로 이동** |

**상단 영역**
- 카테고리 개요 (Gemini가 작성한 2~3줄 요약)
- 트렌드 그래프 (2025 vs 2026 시계열)

---

### [3단계] 소싱 사이트별 추천 상품 리스트

선택한 도매 사이트에서 eBay에 올리기 좋은 상품 50선.

**상품 카드 구성**
- 상품명 (한글 + 영문)
- 썸네일 이미지
- 도매가 (KRW)
- 예상 eBay 판매가 (USD)
- 예상 마진율 (%)
- 무게/부피 (국제 배송비 산정용)
- eBay 카테고리 ID
- 추천 영문 SEO 키워드 (3~5개)
- 도매 사이트 원본 링크
- (향후) "Magic One 상세페이지 생성" / "NextOne 자동 등록" 버튼

## 4. Gemini 리서치 연동 (하루 1회 배치)

### 작동 방식

```
매일 정해진 시각 (예: 새벽 4시)
    ↓
Gemini가 정해진 프롬프트로 리서치 실행
    ↓
결과를 정해진 JSON 스키마로 출력
    ↓
사이트의 /data/{날짜}/ 폴더에 JSON 파일로 저장 (스냅샷)
    ↓
사이트는 사용자가 선택한 기간/날짜의 JSON을 읽어서 표시
```

**핵심**: 실시간 API 호출 없음. 하루 1회 정적 데이터 갱신 → 빠르고 안정적.

### "기간별 분류" + "매일 갱신" 처리 방법

두 요구사항은 충돌하지 않습니다. **데이터 모델을 "스냅샷"과 "뷰"로 분리**하면 됩니다.

```
[매일 쌓이는 것 — 재료]    =  스냅샷
   Gemini가 매일 1회 → 그 날짜의 랭킹 JSON 1세트 저장

[화면에 보이는 것 — 뷰]    =  사용자가 기간 선택
   ├ "오늘"          → 가장 최근 스냅샷 1개
   ├ "2026년"        → 2026년 스냅샷들 중 최신 (또는 분기 평균)
   ├ "2025년"        → 2025년 결산 고정 스냅샷
   └ "비교 보기"      → 2025 vs 2026 나란히
```

**구체적 처리**

| 기간 | 갱신 방식 | 데이터 저장 위치 |
|---|---|---|
| **2025년 (과거)** | Gemini 1회 리서치 후 고정 | `/data/2025-final/` |
| **2026년 (현재)** | 매일 갱신, 최신 스냅샷 표시 | `/data/2026-MM-DD/` |
| **트렌드 그래프** | 누적된 스냅샷 자동 활용 | 여러 날짜 폴더 집계 |

**화면 UI**

모든 페이지 상단에 기간 선택 토글:
```
[ 2025년 ]  [ 2026년 (최신) ]  [ 비교 보기 ]
```

### 변동 표시 (Delta) 정책

매일 스냅샷이 쌓이므로, 사용자가 "어제/지난주/지난달 대비 얼마나 바뀌었는지"를 직관적으로 볼 수 있게 표시.

| 표시 위치 | 비교 기준 | 표시 방식 |
|---|---|---|
| 카테고리 카드 | 어제 vs 오늘 | ▲3 (3계단 상승) / ▼2 / ─ (변동 없음) |
| 브랜드 랭킹 | 7일 전 vs 오늘 | ▲ 상승 / ▼ 하락 / NEW (신규 진입) |
| 상품 랭킹 | 7일 전 vs 오늘 | ▲ / ▼ / NEW / HOT (3일 연속 상승) |
| 트렌드 그래프 | 최근 30일 추이 | 라인 차트 |

**계산 로직**
```
1. Gemini가 오늘 스냅샷 생성 → /data/2026-05-13/categories.json 저장
2. 사이트 로드 시:
   - 오늘 데이터 + 어제 데이터(/data/2026-05-12/) 동시 로드
   - 같은 entity_id의 rank 차이 계산 → delta 필드 주입
3. 화면에 ▲▼ 아이콘으로 렌더링
```

**기준 날짜 비교 우선순위**
- 어제 데이터가 없으면 → 그제 → 가장 가까운 과거 스냅샷 사용
- 7일 전 비교 시 데이터 없으면 → 가장 가까운 ±2일 이내 스냅샷

### 스냅샷 압축 (Retention) 정책

매일 신규 폴더가 생기면 1년에 365개, 2년이면 730개로 폭증 → 저장공간/로딩속도 문제 발생.
**시간이 오래된 데이터일수록 해상도를 낮춰 압축**.

| 기간 | 보관 단위 | 예시 |
|---|---|---|
| **최근 30일** | 일간 (매일) | `2026-04-13/` ~ `2026-05-12/` 매일 보관 |
| **30일 ~ 180일** | 주간 (월요일 1개) | `2026-W12/`, `2026-W13/` ... |
| **180일 ~ 1년** | 월간 (월초 1개) | `2026-01/`, `2026-02/` ... |
| **1년 이상** | 분기/연간 | `2025-Q4/`, `2025-final/` |

**압축 동작 방식**
```
매일 새 스냅샷 저장 후, 압축 스크립트 자동 실행:

1. 30일 지난 일간 폴더 → 해당 주의 월요일 스냅샷만 남기고 나머지 삭제
2. 180일 지난 주간 폴더 → 해당 월의 1주차 스냅샷만 남기고 나머지 삭제
3. 1년 지난 월간 폴더 → 분기 대표 1개만 남기고 나머지 삭제
```

**장점**
- 저장공간 절약 (1년 후 폴더 수: 365 → 약 60개로 감소)
- 최근 데이터는 정밀, 과거 데이터는 추세 파악용으로 충분
- 트렌드 그래프 로딩 속도 유지

**구현 위치**: `scripts/compact-snapshots.ts` — 일일 리서치 스크립트 마지막 단계에서 실행

## 5. 데이터 구조

### JSON 파일 구조 (가장 단순한 방식)

```
/data
  /2025-final/                ← 2025년 결산 (1회 리서치 후 고정)
    categories.json
    /categories/
      beauty.json
      stationery.json
      ...
    /sourcing/
      beauty_cheonyu.json
      ...
      
  /2026-05-12/                ← 매일 신규 폴더 (오늘의 스냅샷)
    categories.json            ← 1단계 데이터
    /categories/
      beauty.json              ← 2단계 데이터 (카테고리별)
      stationery.json
      ...
    /sourcing/
      beauty_cheonyu.json      ← 3단계 데이터 (카테고리+소싱처)
      beauty_perzoom.json
      ...
      
  /2026-05-13/                ← 다음날 또 신규 폴더
  /2026-05-14/
    ...
    
  /latest.json                ← {"date": "2026-05-14"} 최신 날짜 포인터
```

**기간 선택 시 동작**
- "2025" 선택 → `/data/2025-final/` 읽기
- "2026" 또는 "오늘" 선택 → `latest.json` 확인 후 해당 날짜 폴더 읽기
- "비교 보기" → 두 폴더 동시 로드해서 나란히 표시

**장점**
- DB 없이 시작 가능
- Git으로 데이터 버전 관리 가능
- 일자별 히스토리 자연스럽게 보존됨
- 추후 트렌드 그래프 만들 때 과거 데이터 그대로 활용

### (확장) DB 전환 시 스키마

```sql
categories (id, name, rank, trend, competition, margin, zone, updated_at)
brands (id, category_id, name, rank, country)
products (id, category_id, brand_id, name, rank, ebay_price_usd)
sourcing_sites (id, category_id, name, url, rank, reliability)
sourcing_products (id, sourcing_site_id, category_id, name_kr, name_en,
                   wholesale_price_krw, ebay_price_usd, margin_pct,
                   weight_g, ebay_category_id, seo_keywords[])
rankings_history (entity_type, entity_id, rank, date)  -- 트렌드용
```

## 6. 기술 스택

| 영역 | 선택 | 이유 |
|---|---|---|
| 빌드 도구 | **Vite** | 빠른 HMR, 가벼움, 정적 사이트 빌드에 최적 |
| 프론트엔드 | **React 18 + TypeScript** | 타입 안전성 + 데이터 스키마 보호 |
| 스타일 | **Plain CSS + CSS Variables** | 디자인 시스템이 이미 CSS 변수로 잘 짜여있어 Tailwind 불필요 |
| 폰트 | Pretendard + JetBrains Mono (CDN) | 한국어 가독성 + 숫자/순위 모노스페이스 |
| 차트 | **자체 SVG 컴포넌트** | Recharts 불필요, 가벼운 스파크라인/트렌드선 자체 구현 |
| 데이터 저장 | JSON 파일 (1차) → Supabase (확장) | 시작은 단순하게, 확장은 자연스럽게 |
| 리서치 | Gemini (수동 또는 일 1회 자동) | 사용자 요구사항 |
| 자동화 | GitHub Actions 또는 Vercel Cron | 무료 티어로 일 1회 실행 충분 |
| 배포 | Vercel / Netlify | Vite 정적 빌드 그대로 호스팅 |
| 결제(추후) | 토스페이먼츠 또는 Stripe | 한국/글로벌 모두 대응 |

**Next.js를 안 쓴 이유**: 이 사이트는 SSR이 꼭 필요한 동적 페이지가 없고, Gemini가 만든 JSON을 정적으로 보여주는 구조라 Vite SPA가 더 가볍고 빠릅니다. SEO가 중요해지면 추후 Next.js로 마이그레이션 가능.

**Tailwind를 안 쓴 이유**: 기존 디자인 시스템의 CSS 변수(`--red`, `--blue`, `--green` 등)가 oklch 색공간으로 정밀하게 짜여 있어, Tailwind로 옮기면 오히려 디자인 토큰이 분산됩니다.

## 7. eBay 특화 기능 (차별화 포인트)

큐텐 버전과 달리 eBay는 **글로벌 + 영문 SEO + 국제 배송**이 핵심 변수.

- **국제 배송비 계산기**: 무게/부피 입력 → ePacket / K-Packet / EMS 비교
- **eBay 수수료 자동 차감**: 카테고리별 final value fee 반영한 실 마진 표시
- **영문 SEO 키워드**: Gemini가 카테고리별 high-volume 키워드 추출
- **HS Code 매칭** (전문 셀러용): 통관용 코드 자동 매칭

## 9. MVP 개발 로드맵

### 1차 MVP (2주) — 정적 사이트
- [ ] Gemini로 1회 리서치 수동 실행 → JSON 파일 생성
- [ ] 1단계 메인 페이지 (카테고리 랭킹 20개)
